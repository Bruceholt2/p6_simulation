"Create a document called docs/xer_format_reference.md that describes the P6 XER file format in detail. Cover the following:

XER files are text files with tab-delimited data
%T lines define table names (e.g., %T TASK, %T TASKPRED)
%F lines define field names for the table that follows
%R lines are data rows for the current table
The key tables we need are:

PROJECT: project-level data (proj_id, proj_short_name, last_recalc_date)
CALENDAR: calendar definitions (clndr_id, clndr_name, clndr_data — which contains a structured string of work hours per day and exceptions)
TASK: activities (task_id, proj_id, task_code, task_name, task_type, status_code, total_float_hr_cnt, orig_dur_hr_cnt, remain_dur_hr_cnt, target_start_date, target_end_date, early_start_date, early_end_date, late_start_date, late_end_date, clndr_id, phys_complete_pct)
TASKPRED: relationships (task_pred_id, task_id, pred_task_id, pred_type, lag_hr_cnt)
TASKRSRC: resource assignments (taskrsrc_id, task_id, rsrc_id, target_qty_per_hr, target_cost)
RSRC: resource definitions (rsrc_id, rsrc_name, rsrc_short_name, rsrc_type)
RSRCRATE: resource capacity (rsrc_rate_id, rsrc_id, max_qty_per_hr, Cost_per_qty, start_date)


Include notes about date format (typically yyyy-MM-dd HH:mm), encoding issues (Latin-1 common), and how the clndr_data string is structured."
